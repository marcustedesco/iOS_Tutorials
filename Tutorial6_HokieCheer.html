Step 0: Earlier Phases of the Software Life Cycle

We assume that earlier phases of the Software Life Cycle have been completed. See Dr. Balci's Software Life Cycle. The life cycle processes Problem Formulation, Requirements Engineering, Architecting, and Design are assumed to be completed. In this tutorial, we focus on the Programming process.

Step 1: Application Functionality Specification

The HokieCheer app provides four main functionalities as described below.

Shake the iOS device to play the turkey gobble sound.
Play / Pause / Stop the sound of keys for key play.
Play / Pause / Stop the Tech Triumph song.
Play / Pause / Stop the Hokie Pokie song.

Note that the iOS Simulator's Hardware → Shake Gesture does not work! You need to deploy the app to an iOS test device for the shake gesture to work.

Step 2: Creation of a New Project

  1. Launch Xcode. Create a new project by selecting File → New → Project...
  2. Select iOS → Application → Tabbed Application template. Click Next.
  3. In the Choose options for your new project dialog,

    • Enter Product Name as HokieCheer.
    • Enter Your Name as Organization Name.
    • Enter com.yourname as your company/organization unique identifier. Note that domain names, e.g., apple.com, ibm.com, microsoft.com, are unique.
    • The string com.yourname.HokieCheer becomes the Bundle ID for your application uniquely identifying it in the App Store for distribution.
    • Select Swift as the programming language to use.
    • Select iPhone for the Devices, implying that your app will run on those iPhone and iPod Touch devices that can run iOS 8.0 or higher. Note that some old iOS devices cannot be upgraded to version 8.0.
    • Click Next. In the File Browser window displayed,
      1. Select a location on your hard disk to store your project.
      2. Uncheck Source Control to disable version control. Click Create to create your project.

Step 5: Linking the Frameworks Needed

Our app plays:

  • a short (30 seconds or shorter) sound when the device is shaken, requiring the AudioToolbox.framework.
  • long songs with play-pause-stop buttons, requiring AVFoundation.framework.

To add a framework to your project so that its classes are made available and linked to your application:

  1. In Xcode Project Navigator, click HokieCheer on the top, select the General tab, and expand Linked Frameworks and Libraries.
  2. Click the + icon to bring up the list of frameworks.
  3. Command click AudioToolbox.framework and AVFoundation.framework to select them. Click Add to add them to your project.
  4. Right click the HokieCheer group and select New Group
  5. Name the new Group as Frameworks and move the newly added framework files into it.

Turkey Gobble View Controller Class Specification

To play the Turkey Gobble short (≤ 30 secs) sound when the user applies the Shake Gesture to the device, we will use the System Sound Services under the AudioToolbox framework in the Media Layer as shown in the following screenshot.



“System Sound Services provides a C [programming lnaguage] interface for playing short sounds and for invoking vibration on iOS devices that support vibration. You can use System Sound Services to play short (30 seconds or shorter) sounds.” [Apple]

Copy and paste the documented code given below. Carefully study the code, understand what it is doing, and learn from it!

TurkeyGobbleViewController.swift file:

//

//  TurkeyGobbleViewController.swift

//  HokieCheer

//

//  Created by Osman Balci on 8/8/14.

//  Copyright (c) 2014 Osman Balci. All rights reserved.

//

 

import UIKit

import AudioToolbox

 

class TurkeyGobbleViewController: UIViewController {

 

    var turkeyGobbleSoundID: SystemSoundID = 0

   

    /*********************************************************

    MARK: - viewDidLoad Method: Create the System Sound Object

    *********************************************************/

   

    override func viewDidLoad() {

        super.viewDidLoad()

 

        // Obtain the file path of type NSURL to the turkey gobble sound file, turkeyGobble.wav.

        var soundFilePath: NSURL? = NSBundle.mainBundle().URLForResource("turkeyGobble", withExtension: "wav")

        

        /*

        The CFURL (Core Foundation Uniform Resource Locator) "opaque type provides facilities for

        creating, parsing, and dereferencing URL strings. CFURL is useful to applications that need

        to use URLs to access resources, including local files." [Apple]

 

        CFURLRef is a C programming language type and is provided in Core Foundation, which

        provides code in lower-level C framework.

       

        Type Cast the NSURL type to be a C pointer type CFURLRef

        */

        var soundFileURLRef = soundFilePath! as CFURL

       

        /*

        Create a system sound object for the short sound (30 seconds or shorter) in file turkeyGobble.wav

       

        Load the turkey gobble sound file into memory from its file and assign its identifier

        to the turkeyGobbleSoundID instance variable.

       

        & is a unary operator known as the Address Operator. It basically represents

        two-level indirection. &turkeyGobbleSoundID points to another pointer that contains

        the memory address of the sound; hence, double pointers or double indirection.

       

        AudioServicesCreateSystemSoundID is a C programming language function, which takes the URL of the

        audio file to play, soundFileURLRef, as input and provides the ID, &turkeyGobbleSoundID, of the

        created system sound object associated with the specified audio file, turkeyGobble.wav.

        */

        AudioServicesCreateSystemSoundID(soundFileURLRef, &turkeyGobbleSoundID)

    }

   

    /******************************************************************

    MARK: - Shake Gesture Detection and Playing the Turkey Gobble Sound

    ******************************************************************/

   

    /*

    The TurkeyGobbleViewController object must be the first responder to be able to receive

    and handle motion events. The following two instance methods are used for this purpose.

    */

   

    override func canBecomeFirstResponder() -> Bool {

        // Indicate that the TurkeyGobbleViewController object can be the first responder

        return true

    }

   

    override func viewDidAppear(animated: Bool) {

        super.viewDidAppear(animated)

       

        // Upon the view appearing, assign the TurkeyGobbleViewController object to be the first responder

        self.becomeFirstResponder()

    }

 

    // This method plays the turkey gobble sound if the detected motion is a Shake Gesture

    override func motionEnded(motion: UIEventSubtype, withEvent event: UIEvent) {

       

        // If the detected motion is a Shake Gesture

        if event.subtype == UIEventSubtype.MotionShake {

           

            // Then, play the turkey gobble sound

            AudioServicesPlaySystemSound(turkeyGobbleSoundID)

        }

    }

 

}

Key Play View Controller Class Specification

Copy and paste the documented code given below. Carefully study the code, understand what it is doing, and learn from it!

KeyPlayViewController.swift file:

//

//  KeyPlayViewController.swift

//  HokieCheer

//

//  Created by Osman Balci on 8/8/14.

//  Copyright (c) 2014 Osman Balci. All rights reserved.

//

 

import UIKit

import AVFoundation

 

class KeyPlayViewController: UIViewController {

 

    /*

    Declare audioPlayer as an Optional instance variable (property) to point to an

    AVAudioPlayer object. The initial value of nil implies that its value is absent.

    */

    var audioPlayer: AVAudioPlayer? = nil

   

    /****************************

    MARK: - viewWillAppear Method

    ****************************/

   

    /*

    Just before the view becomes visible, create an AVAudioPlayer object for playing the keyPlay.wav

    audio file and store its obj ref into the audioPlayer instance variable declared above.

    */

    override func viewWillAppear(animated: Bool) {

        super.viewWillAppear(animated)

       

        // Obtain the file path of type NSURL to the audio file keyPlay.wav.

        var audioFileURL: NSURL? = NSBundle.mainBundle().URLForResource("keyPlay", withExtension: "wav")

       

        /*

        Create an AVAudioPlayer object for playing the keyPlay.wav audio file.

        If the object creation is successful, the Optional local variable newAudioPlayer

        will have a value; otherwise, it will be nil implying that it has no value.

        */

        var newAudioPlayer: AVAudioPlayer? = AVAudioPlayer(contentsOfURL: audioFileURL!, error: nil)

       

        // Unwrap the Optional newAudioPlayer. If it has a value, assign it to createdAudioPlayer.

        // Otherwise, assign nil implying that it has no value.

        if let createdAudioPlayer = newAudioPlayer {

           

            // An AVAudioPlayer object is successfully created. Assign its obj ref value to audioPlayer

            audioPlayer = createdAudioPlayer

           

        } else {

            // AVAudioPlayer object creation failed!

           

            // Misspell the filename for URLForResource above to see this error message.

            println("AVAudioPlayer object creation failed!")

        }

    }

   

    /*************************************

    MARK: - Play, Pause, and Stop Methods

    *************************************/

   

    @IBAction func playAudio(sender: UIBarButtonItem) {

       

        // Calling the play method implicitly calls the prepareToPlay method

        // if the audio player is not already prepared to play.

        audioPlayer!.play()

    }

   

    @IBAction func pauseAudio(sender: UIBarButtonItem) {

       

        // Pause playback. The audio remains ready to resume playback from

        // where it left off when the Play button is tapped.

        audioPlayer!.pause()

    }

   

    @IBAction func stopAudio(sender: UIBarButtonItem) {

        /*

        The stop method stops the playback of the audio file, but it does not reset the audio file

        to start from the beginning, the next time the Play button is tapped. Therefore, if you tap

        Play after you tap Stop, the audio will resume playing from the point where it left off.

        */

        audioPlayer!.stop()

       

        // Set the playback point in time to 0 second; that is, bring the audio to its beginning.

        audioPlayer!.currentTime = 0

    }

   

    /*******************************

    MARK: - viewWillDisappear Method

    *******************************/

   

    override func viewWillDisappear(animated: Bool) {

        super.viewWillDisappear(animated)

       

        // Just before the view is dismissed, covered, or otherwise hidden, stop the audio playing.

        audioPlayer!.stop()

       

        // Set the playback point in time to 0 second; that is, bring the audio to its beginning.

        audioPlayer!.currentTime = 0

    }

   

    /********************

    MARK: - Other Methods

    ********************/

   

    override func viewDidLoad() {

        super.viewDidLoad()

        // Do any additional setup after loading the view.

    }

 

    override func didReceiveMemoryWarning() {

        super.didReceiveMemoryWarning()

        // Dispose of any resources that can be recreated.

    }

   

}
