Upon launch, the FlashLight app turns the iOS device into a light source.

Step 2: Creation of a New Project

  1. Launch Xcode. Create a new project by selecting File → New → Project... (The three dots after a menu item, referred to as ellipsis, imply that additional options will be presented to the user after the selection of the menu item.)
  2. Select iOS → Application → Single View Application template. Click Next.



  3. In the Choose options for your new project dialog, as shown in the following screenshot,

    • Enter Product Name as FlashLight. Note that the product name is your app name displayed under the app icon on the home screen of the iOS device. Therefore, it should be selected to be short enough to fit into the available space so that it is not truncated. The app name can have spaces such as Google Maps.
    • Enter Your Name as Organization Name
    • Enter com.yourname as your company/organization unique identifier. Note that domain names, e.g., apple.com, ibm.com, microsoft.com, are unique.
    • The string com.yourname.FlashLight becomes the Bundle ID for your application uniquely identifying it in the App Store for distribution.
    • Select Swift as the programming language to use. Swift requires iOS version 7.0 or higher.
    • Select Universal for the Devices, implying that your app will run on those iOS devices, i.e., iPhone, iPad, and iPod Touch, that can run iOS 7.0 or higher. Note that some old iOS devices cannot be upgraded to version 7.0.



Click Next. In the File Browser window displayed,

  1. Select a location on your hard disk to store your project.
  2. Uncheck Source Control to disable version control. Click Create to create your project.

Step 4: iOS App Icons and Launch Images Creation Guidelines

    • Go to iOS Human Interface Guidelines for iOS 7. Read through the guidelines to familiarize yourself.
    • Click Icon and Image Design on the bottom of the left menu.
    • Click App Icon under Icon and Image Design on the left menu to learn about how to properly create an iOS app icon.
    • Click Launch Images under Icon and Image Design on the left menu to learn about how to properly create iOS launch images.

Here are some important guidelines for iOS app icon creation:

  1. Use Adobe Illustrator to create an iOS icon as a vector-based illustration so that you can resize without losing quality. Do not use bitmap-based image formats such as GIF, JPEG or TIFF. Search the web for vector-based illustrations to manipulate in Adobe Illustrator to create your icon.
  2. Select "pixels" as the unit of measurement in Adobe Illustrator Preferences. You need to work with "pixels" since the icon sizes are given in that unit.
  3. Resize your icon according to the size desired in Adobe Illustrator without losing image quality.
  4. After resizing, export your vector-based illustration as a PNG (Portable Network Graphics) file to preserve the vector-based representation. Always use PNG for all your images in iOS applications.
  5. Create the icon as a square image with full body according to the size specified. The iOS icons have rounded corners; however, you do not create your icon with rounded corners. Apple automatically creates the rounded corners by masking your square icon image.

    Square App Icon Image
    Prepared by You    		 App Icon with Rounded Corners
    Applied by Apple     by Masking

  6. Never use Transparency for your iOS icons. Do not use "Interlaced" when generating the PNG file.
  7. Use 72 dpi for image resolution. Apple indicates that "The standard bit depth for icons and images is 24 bits—that is, 8 bits each for red, green, and blue—plus an 8-bit alpha channel."

Step 5: Setting General Project Attributes

Set the following project attributes under the General tab as shown in the following screenshot:

  1. Click the project name FlashLight in the project navigator to display the General project settings as shown in the following screenshot.
  2. Click the Team pop-up menu and select Add an Account...
  3. Enter your Apple ID and password to attach your iOS Developer Program account to Xcode. You must be a registered iOS Developer.
  4. If you get an error message about "No provisioning profile...", click Fix Issue to fix the problem.
  5. Select your Team so that you can deploy your app to iOS test devices for testing purposes.

  6. Select Upside Down for Device Orientation for all iOS devices.
  7. Select Hide status bar under Status Bar Style since we do not want to show the status bar.



Step 6: Application Icons and Launch Images

    1. Download the FlashLight_files.zip file.
    2. Uncompress it.
    3. Click Images.xcassets in the project navigator as shown below to display the Image Assets Pane.
    4. Drag and drop the dowloaded folders App Icons and Launch Images into the Image Assets Pane as shown below.
    5. Images dragged and dropped in the Image Assets Pane are stored in the Media Library.



Setting the App Icons

  1. Click AppIcon in the Image Assets Pane as shown in the following screenshot.
  2. Select all options in the Attributes Inspector on the top right corner of the window as shown below.
  3. Click the Media Library icon in the Library Selector Bar to show all of the image assets for the project.
  4. Click the first app icon on the top of the listing. Then while pressing the Shift key, use the Down-Arrow key to select all of the remaining app icons.
  5. Drag and drop the selected app icons into the AppIcon assets area as shown in the following screenshot.
  6. Drag and drop the app icons on the remaining empty placeholders with corresponding sizes.
  7. Make sure that each placeholder has an image as shown below.
  8. You can inspect the size, filename, and other attributes of a selected app icon using the Attributes Inspector as depicted in the screenshot below.


Setting the Launch Images

  1. Click LaunchImage in the Image Assets Pane as shown in the following screenshot.
  2. Select all options in the Attributes Inspector on the top right corner of the window as shown below.
  3. Click the Media Library icon in the Library Selector Bar to show all of the image assets for the project.
  4. Click the first Launch Image. Then while pressing the Shift key, use the Down-Arrow key to select all of the remaining launch images.
  5. Drag and drop the selected launch images into the LaunchImage assets area as shown in the following screenshot.
  6. Drag and drop the launch images on the remaining empty placeholders with corresponding sizes.
  7. Make sure that each placeholder has an image as shown below.
  8. You can inspect the size, filename, and other attributes of a selected image using the Attributes Inspector as depicted in the screenshot below.



Step 7: Running the App under the iOS Simulator

    1. Make sure that all of your files are saved.
    2. Click the device selection pop-up menu for the iOS Simulator on the top left corner of the Xcode window and select a device as shown in the following screenshot.
    3. Click the Run button on the top left corner of the Xcode window to execute your app under the iOS Simulator.
